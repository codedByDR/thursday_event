Database Schema Overview

The STHENIA application uses a comprehensive 20-table database schema designed to support enterprise employee wellbeing and AI-powered request management.

Key Entity Categories

  • Core Identity & Requests: USERS, REQUESTS, APPROVALS
  • Wellbeing & Mental Health: WELLBEING_CHECKINS, COUNSELLING_SESSIONS, COUNSELLING_MESSAGES
  • Security & Compliance: HARASSMENT_REPORTS, AUDIT_LOGS, POLICIES
  • Personal Assistant: GROCERY_ITEMS, ALARMS, TODOS, MEAL_PLANS
  • Workplace Safety: SHIFT_LOGS, MATERNITY_PLANS
  • Learning & Development: LEARNING_MODULES, LEARNING_PROGRESS
  • AI & Communication: KIRA_SESSIONS, KIRA_MESSAGES, NOTIFICATIONS, EMAIL_QUEUE

Security Features

  • Encrypted sensitive fields in HARASSMENT_REPORTS
  • Anonymous reporting with SHA-256 hashed identifiers
  • Comprehensive audit logging for AI decisions
  • Magic token system for secure email approvals
  • Role-based access control (Employee, Supervisor, HR, Admin)
